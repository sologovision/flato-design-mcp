# Agent Entry

This repository uses `.harness/` for AI development workflow and
`.project-wiki/` for durable project knowledge.

Start here:

1. Read `.harness/index.md`.
2. Use `.harness/catalog.md` to route the task.
3. Read only the relevant `.project-wiki/` pages.
4. Verify claims against current files before changing code or docs.

Keep changes small, verified, and directly tied to the user goal.

## Shared Flato MCM

For public Flato positioning, README/registry copy, installation narrative,
agent-facing skill guidance, examples, docs, sales/marketing language, and
customer-facing MCP descriptions, read the shared MCM project
`flato.marketing` before drafting or editing. Use explicit `project_id` for
every MCM operation.

Do not create repo-scoped marketing cognition for this distribution repository.
Durable public/runtime cognition belongs in `flato.marketing`; MCP tool
contracts, package layout, OAuth behavior, release notes, and internal
maintenance knowledge belong in `.project-wiki/`.
