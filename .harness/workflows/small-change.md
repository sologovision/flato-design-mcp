# Small Change Workflow

Use for narrow, reversible changes to docs, examples, metadata, or skill text.

1. Route the task through `.harness/catalog.md`.
2. Read the relevant wiki pages and current source files.
3. Make the smallest edit that satisfies the user goal.
4. Check for duplicated guidance across README, docs, skill, plugin skill, and
   registry metadata when the changed fact is public.
5. Verify with targeted commands.
6. Update `.project-wiki/` only when the change affects durable behavior,
   contracts, decisions, or repeated workflow knowledge.
