# Large Change Workflow

Use when a task changes multiple public surfaces, installation flow, MCP
operating contract, package layout, or release metadata.

1. Summarize the intended product outcome and compatibility risks.
2. Inspect all routed files in `.harness/catalog.md`.
3. Identify duplicated claims that must stay in sync.
4. Make changes in small batches.
5. Verify each public surface after editing.
6. Update `.project-wiki/` and `.harness/update-log.md`.
7. Report remaining uncertainty instead of encoding guesses.
