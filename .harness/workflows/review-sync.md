# Review Sync Workflow

Use for platform review, marketplace review, security review, or release review
notes.

1. Read existing review notes in `registry/`.
2. Verify public claims against README, docs, plugin manifest, and skill files.
3. Separate reviewer-facing metadata from user-facing installation docs.
4. Keep security and OAuth guidance consistent across all surfaces.
5. Update `.project-wiki/` if the review establishes a durable constraint.
