# Wiki Update Workflow

Use when adding or correcting durable project knowledge.

1. Start from `.project-wiki/schema.md`.
2. Pick the right entity type: architecture, module, feature, contract,
   decision, or method.
3. Cite current repository files as sources.
4. Mark unverified claims as `needs-verification` or keep them in `raw/`.
5. Keep pages short enough for agents to read before implementation.
6. Update `.project-wiki/index.md` when adding a new page.
